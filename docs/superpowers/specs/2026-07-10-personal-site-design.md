# 个人站 + 工具站设计（nbssdlkm.cn）

- 日期：2026-07-10
- 状态：设计已逐节确认，待用户审阅本文档
- 参考台账：Obsidian `工作台账/个人资源/`（MiniPC盘点-2026-07-10、个人VPS与域名、设备清单）

## 1. 背景与目标

- 域名 `nbssdlkm.cn`（阿里云万网注册，Cloudflare DNS，未备案，主域走 CF Tunnel → MiniPC Caddy）。
- 旧站为 Hexo + Stellar，内容陈旧（学生时期定位），整体替换。
- 新站两个使命：**个人品牌主页**（文科/英专转码人设，FicForge 与作品集为重点，同步小红书帖子）+ **工具站**（实用小工具，未来广告商业化）。
- 头号读者：个人品牌受众（同行/未来招聘方），非变现受众。

## 2. 已确认的决策

| 决策 | 内容 |
|---|---|
| 技术栈 | Astro 静态生成，单仓库单项目；线上只有构建产物，服务器不跑任何程序 |
| 设计系统 | 复用 FicForge 的 Ex Libris（羊皮纸/墨绿/鼠尾草/古董金），token 原样抄入；金色保留，与公司「墨蓝+金」乍看可区分即可 |
| 语言 | 纯中文 |
| 站点关系 | 个人站与工具站不解耦：同仓库、同设计系统；工具站挂 `/tools/` 子树，未来可整树迁出独立域名 |
| 部署 | `astro build` → rsync 到 MiniPC `/var/www/html/public`，Caddy 与 CF 隧道零改动 |
| 商业化 | 纯静态保证两条路随时可切：备案线（国内轻量云 + 国内广告联盟）或全球线（CF Pages + AdSense）。现在不选、不预建广告位 |

## 3. 站点结构

| 网址 | 板块 | 内容 |
|---|---|---|
| `/` | 首页 | 一句话人设 + 精选项目 2-3 个（自动读项目清单）+ 最近 3 篇帖子（自动读帖子目录） |
| `/projects` | 作品集 | FicForge 大卡置顶（鼠尾草底+金顶线，视觉锚点），其余项目次级卡片；点击直接跳 GitHub/线上地址 |
| `/posts` | 帖子 | 小红书同步流：列表页 + 详情页，构建时从 md 文件自动生成 |
| `/about` | 关于 | 个人故事（文科转码路径）+ GitHub / 小红书 / 邮箱链接 |
| `/tools` | 工具 | 工具站入口，自动扫描列出所有工具页 |

首页原型已确认（2026-07-10 对话中的 mockup）：顶部五项导航，hero 大字标题，FicForge 深色大卡为唯一视觉焦点。

## 4. 内容模型

- **帖子** `src/content/posts/*.md`：frontmatter 为 `title`、`date`、可选 `source`（小红书原帖链接，页面上标注"首发于小红书"）、可选 `tags`；配图放帖子同目录 `images/`。发帖 = 扔一个 md 文件 + 跑部署脚本。
- **项目清单** `src/data/projects.yaml`：每项含 `name`、`tagline`（一句话）、`url`、`image`、`featured`（是否上首页）与排序。作品集页与首页精选共读此文件，单一事实来源。
- **关于页**：单个页面文件，正文 markdown。
- **工具** `src/pages/tools/*.astro`：每工具一个文件，导出 `title`/`description` 元信息；入口页用 `import.meta.glob` 自动扫描生成目录，加工具零登记。工具逻辑为页内 `<script>`，在访客浏览器运行，无后端、文件不上传。

## 5. 设计系统落地

- `src/styles/exlibris.css`：从 FicForge `src-ui/src/App.css` 抄 `:root` 的颜色/字体变量段，保持变量命名一致（便于双向同步改进）。
- 字体：EB Garamond（display，拉丁字符，自托管 woff2 subset）；中文走 FicForge 同款衬线栈的系统回退（不自托管大体积中文字体，性能优先）。MiniPC Caddy 已有 `/fonts` 字体托管（font-dist，台账记录在用），实现阶段核对其内容后决定复用或替换——此项为唯一待实现期核实项。
- 共享组件：`Base.astro`（页头+导航+页脚）、项目卡、帖子列表项。工具页本体区域保持素净（mono 字体、克制装饰）。

## 6. 仓库与部署

- git 仓库位于项目根（本机 `D:\personal website`，开发主场为 MiniPC，git 同步）。
- 远程：GitHub 公开仓库（站点源码本身是作品集的一部分；用户审阅时可否决改私有）。
- `deploy.sh`：`npm run build` + rsync `dist/` → MiniPC `/var/www/html/public`。旧站成品已在 2026-07-10 清理备份 tar 中，上线即整体替换。
- 附带一次性配置：`@astrojs/rss` 生成帖子 RSS（一次配置零维护，对同行订阅有真实价值）。

## 7. 明确不做（YAGNI）

- 不做双语/i18n（纯中文已定）。
- 不做项目详情页（点卡片直接外跳；哪个项目真需要长文再加）。
- 不做评论、访问统计后端、搜索（流量起来后可加 Cloudflare Web Analytics 一行脚本，免费无 cookie）。
- 不做小红书自动抓取（手动贴 md，频率低不值得自动化）。
- 不预建广告位、不预拆工具站仓库（商业化决策落地时再做）。

## 8. 测试与验证

- 门槛：`npm run build` 零错误零警告。
- 本地 `astro preview` 人工过五个区 + 手机宽度（375px）检查。
- 上线后：主域及五个区 URL 全部 200；帖子详情页随机抽一篇核对渲染。
- 每个工具页上线前：一组已知输入/输出的自检用例（页内 console assert 或最小 test 文件）。

## 9. 原则合规审计（per 全局 CLAUDE.md）

- **复用检查**：设计系统复用 FicForge Ex Libris；部署链复用现有 Caddy + CF 隧道；字体托管复用 `/fonts`（待核对）；项目清单单一数据源，无重复表。
- **无硬编码门**：内容站无业务分类逻辑；广告法红线词扫描对本次范围不适用（无广告文案，内容为用户手写）；工具页不含产品业务规则。
- **测试计划**：见第 8 节。
- **人工确认点**：站点结构、语言、内容模型、首页原型已由用户逐节确认（2026-07-10 对话）；剩余确认点 = 本文档审阅、GitHub 仓库公开性、首批工具清单（追加制，不阻塞骨架）。
